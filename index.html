Tutorial de Bootstrap 5

Desarrollo de Aplicaciones en Red — Actividad 2

En este tutorial voy a intentar explicar como usar Bootstrap desde cero. Cubro desde la instalación hasta crear una pagina web con diseño responsive, componentes y el sistema de grid. La verdad es que me ha costado bastante entender algunas cosas pero creo que al final se entiende bien.

Última actualización: enero 2025  |  Actividad 2 — DAR

1. ¿Qué es Bootstrap?

Bootstrap es un framework de CSS y javascript de código abierto creado originalmente por Twitter en 2010 (creo que fue en ese año, aunque en algunas webs pone 2011). Basicamente su función es darte una serie de estilos y componentes ya hechos para que puedas construir paginas web rapido y con buen aspecto, sin tener que escribir todo el CSS desde cero.

¿Para qué sirve?

  • Crear interfaces web responsive (que se adaptan a movil, tablet y escritorio).
  • Usar componentes visuales ya hechos: botones, formularios, tarjetas, menus, alertas, modales, etc.
  • Aplicar estilos de forma rápida usando clases CSS que ya vienen predefinidas en el framework.
  • Ahorrar mucho tiempo en el front-end de una aplicación.

¿Qué aplicaciones tiene?

Bootstrap se usa en casi cualquier tipo de página web: portfolios, tiendas online, paneles de administración, webs de empresas, etc. Es muy habitual verlo en proyectos que necesitan salir rápido sin invertir mucho tiempo en el diseño. Personalmente lo he visto mucho en proyectos de clase y en tutoriales de youtube.

¿Sobre qué lenguajes se apoya?

  • HTML — Es el lenguaje base. Bootstrap se usa directamente poniendo clases en las etiquetas HTML.
  • CSS — Internamente Bootstrap está escrito en Sass, aunque nosotros usamos directamente el CSS ya compilado (no hace falta saber Sass).
  • JavaScript — Para los componentes interactivos como modales o menus desplegables. En Bootstrap 5 ya no necesita jQuery, que era una libreria que usaban antes (esto lo explica la documentación oficial).

¿Qué prerrequisitos necesita?

  • Saber algo de HTML y CSS (lo basico).
  • Un navegador web moderno (Chrome, Firefox, Edge…).
  • Un editor de código, yo uso Visual Studio Code.
  • Para la versión CDN: conexion a internet.
  • Para npm: tener instalado Node.js (esto es mas avanzado, no hace falta para empezar).
Nota: No hace falta saber JavaScript para usar Bootstrap. La mayoría de los componentes funcionan solo con clases en el HTML, sin escribir nada de JS. (Al menos así me funcionó a mi).

Versiones principales

Versión Año Novedad destacada
Bootstrap 3 2013 Mobile-first por defecto
Bootstrap 4 2018 Flexbox, variables CSS, grid mejorado
Bootstrap 5 ✓ 2021 Sin jQuery, nuevas utilidades, CSS custom properties

En este tutorial usaremos Bootstrap 5.3, que es la version mas reciente. Hay que tener cuidado porque Bootstrap 4 y 5 no son totalmente compatibles, algunas clases cambiaron (me lo encontre cuando seguia tutoriales antiguos de youtube y no me funcionaba nada).

2. Instalación

Bootstrap se puede incluir en un proyecto de varias formas. Las dos mas comunes son: usando un CDN (un enlace externo que carga los archivos de internet) o descargando los archivos a tu ordenador. Para empezar lo mas facil es el CDN, que es lo que voy a usar yo aqui.

Opción A: CDN (recomendada para empezar)

No hace falta instalar nada. Solo hay que añadir dos etiquetas en el HTML: una en el <head> para el CSS y otra antes de cerrar el <body> para el JS. 

<!-- CSS de Bootstrap (va en el <head>) -->
<link href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.2/dist/css/bootstrap.min.css"
      rel="stylesheet">

<!-- JS de Bootstrap (va antes de </body>, importante que sea al final) -->
<!-- nota: si no funciona algo interactivo probablemente es porque no tienes esto -->
<script src="https://cdn.jsdelivr.net/npm/bootstrap@5.3.2/dist/js/bootstrap.bundle.min.js">
</script>
Consejo: El archivo bootstrap.bundle.min.js ya incluye algo que se llama Popper.js por dentro, que necesitan algunos componentes para funcionar (como los menus desplegables). Asi que con ese solo script es suficiente, no hay que añadir nada más.

Opción B: Descarga directa

  1. Ir a getbootstrap.com/download.
  2. Descargar el ZIP con los archivos compilados.
  3. Descomprimir y copiar bootstrap.min.css y bootstrap.bundle.min.js al proyecto.
  4. Enlazarlos con rutas locales en lugar de CDN.
<!-- Rutas locales -->
<link rel="stylesheet" href="css/bootstrap.min.css">
<script src="js/bootstrap.bundle.min.js"></script>

Opción C: npm (para proyectos más avanzados)

Si tienes Node.js instalado, puedes instalar Bootstrap como paquete:

npm install bootstrap@5.3.2

Y luego importarlo en tu archivo JS principal:

import 'bootstrap/dist/css/bootstrap.min.css';
import 'bootstrap/dist/js/bootstrap.bundle.min.js';

¿Se necesita base de datos?

No. Bootstrap es un framework de front-end puro: solo afecta a la capa visual de la página. No tiene ninguna relación con bases de datos ni con el servidor.

¿Necesita variables de entorno?

No en la forma básica de uso. Solo si usas herramientas de construcción como Webpack o Vite con Sass necesitarías configurar rutas de importación en el archivo de configuración del proyecto.

Versiones necesarias

  • Node.js: 14 o superior (solo si se usa npm).
  • Navegador: cualquier navegador moderno compatible con ES6 y CSS Grid.
  • Bootstrap 5.x no funciona correctamente en Internet Explorer.

3. Primeros pasos: Hola Mundo

Aqui vamos a hacer la primera prueba. Yo recomiendo abrir el VSCode y crear un archivo nuevo que se llame index.html. Luego copias esta estructura y ya puedes empezar a jugar con las clases.

Estructura minima de un archivo HTML con Bootstrap

<!DOCTYPE html>
<html lang="es">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Mi primera página con Bootstrap</title>

  <!-- esto carga el CSS de bootstrap, sin esto no funciona nada -->
  <link href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.2/dist/css/bootstrap.min.css"
        rel="stylesheet">
</head>
<body>

  <div class="container">
    <h1 class="text-center mt-5">¡Hola Mundo con Bootstrap!</h1>
    <p class="text-center text-muted">Esta es mi primera página con Bootstrap.</p>
    <div class="text-center">
      <button class="btn btn-primary">Haz clic aquí</button>
    </div>
  </div>

  <!-- el JS va siempre al final, antes del cierre del body -->
  <script src="https://cdn.jsdelivr.net/npm/bootstrap@5.3.2/dist/js/bootstrap.bundle.min.js">
  </script>
</body>
</html>

¿Qué hace cada clase?

  • container — Añade margenes laterales y centra el contenido. Hay tambien container-fluid que ocupa todo el ancho (yo al principio los confundia).
  • text-center — Centra el texto.
  • mt-5 — Añade margen arriba (margin-top). El número va del 0 al 5.
  • text-muted — Pone el texto en gris clarito.
  • btn btn-primary — Convierte el elemento en un boton azul de Bootstrap. Siempre hay que poner las dos clases juntas.
Importante: No olvidarse de la etiqueta <meta name="viewport"...>. Si no la pones el responsive no funciona bien en el movil. A mi me paso y no entendia porque la pagina se veia rara en el telefono.

¿Cómo ver el resultado?

  1. Guardar el archivo como index.html.
  2. Abrirlo en el navegador haciendo doble clic (o arrastrarlo a Chrome).
  3. Si usas Visual Studio Code, hay una extension que se llama Live Server que recarga la pagina sola cada vez que guardas. La recomiendo mucho.

4. Utilización del framework

¿Cómo empezar a crear una aplicación base?

Para empezar un proyecto con Bootstrap desde cero los pasos son bastante sencillos. Basicamente hay que crear un archivo HTML con la estructura minima y a partir de ahi ir añadiendo componentes.

Estos serian los pasos que yo sigo:

  1. Crear una carpeta para el proyecto (por ejemplo mi-proyecto/).
  2. Dentro, crear el archivo index.html.
  3. Pegar la estructura minima con los enlaces al CDN de Bootstrap (como se vio en la seccion anterior).
  4. Dentro del <body> añadir un <div class="container"> como contenedor principal.
  5. Desde ahi ya se pueden ir añadiendo filas (row), columnas (col) y componentes.

La estructura de carpetas de un proyecto sencillo con Bootstrap quedaria algo asi:

mi-proyecto/
├── index.html          <-- archivo principal
├── css/
│   └── estilos.css     <-- CSS propio (opcional, si quieres personalizar)
└── img/
    └── foto.jpg        <-- imagenes que uses
Nota: Si usas solo el CDN no hace falta la carpeta css/ para Bootstrap, porque los archivos se cargan de internet. Yo la uso cuando quiero añadir mis propios estilos encima.

¿Cómo crear una primera página con Bootstrap paso a paso?

Aparte del Hola Mundo de la seccion anterior, para tener una "aplicacion base" mas completa lo normal es partir de esta plantilla y ir rellenando: 

<!DOCTYPE html>
<html lang="es">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Mi App</title>
  <link href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.2/dist/css/bootstrap.min.css" rel="stylesheet">
  <!-- si tienes CSS propio lo enlazas despues del de Bootstrap -->
  <link rel="stylesheet" href="css/estilos.css">
</head>
<body>

  <!-- 1. Navbar arriba -->
  <nav class="navbar navbar-expand-lg bg-dark navbar-dark">
    <div class="container">
      <a class="navbar-brand" href="#">MiApp</a>
    </div>
  </nav>

  <!-- 2. Contenido principal -->
  <main class="container mt-4">
    <div class="row">
      <div class="col-md-8">
        <h1>Titulo principal</h1>
        <p>Contenido de la pagina...</p>
      </div>
      <div class="col-md-4">
        <!-- barra lateral -->
      </div>
    </div>
  </main>

  <!-- 3. Footer abajo -->
  <footer class="bg-dark text-white text-center py-3 mt-5">
    <p class="mb-0">Mi App © 2025</p>
  </footer>

  <script src="https://cdn.jsdelivr.net/npm/bootstrap@5.3.2/dist/js/bootstrap.bundle.min.js"></script>
</body>
</html>

¿Cómo visualizar los resultados de la ejecución?

Bootstrap no es como un programa que se "ejecuta", es simplemente un archivo HTML que se abre en el navegador. No hay que compilar nada ni arrancar ningun servidor (a no ser que el proyecto use Node.js o similar). Las formas de verlo son:

  • Doble clic en el archivo .html — la opcion mas simple. Se abre directamente en el navegador. El problema es que cada vez que guardas hay que refrescar manualmente con F5.
  • Live Server (extension de VSCode) — la que yo recomiendo. Se instala en VSCode, haces clic en "Go Live" abajo a la derecha, y la pagina se recarga automaticamente cada vez que guardas el archivo.
  • Herramientas de desarrollador del navegador (F12) — muy util para ver como queda en distintos tamaños de pantalla. En Chrome hay un icono de movil arriba a la izquierda del inspector que simula distintos dispositivos.
Consejo — vista responsive en Chrome

Para simular como se ve la pagina en movil sin tener que usar el telefono:

  1. Abre el archivo en Chrome
  2. Pulsa F12 para abrir las herramientas de desarrollador
  3. Haz clic en el icono de movil/tablet que aparece arriba a la izquierda
  4. Selecciona el dispositivo que quieras simular en el desplegable

5. Explicación del funcionamiento del framework

¿Actúa sobre páginas, ficheros, secciones...?

Bootstrap actua principalmente a nivel de ficheros HTML y secciones dentro de esos ficheros. No es un framework que genere paginas automaticamente ni que tenga una estructura de rutas como podria tener Next.js o Laravel. Simplemente añades sus clases a las etiquetas HTML que ya tienes y el framework se encarga de aplicarles los estilos.

Dicho de otra forma: Bootstrap no sabe ni le importa como esta organizado tu proyecto. Da igual si tienes un solo archivo HTML o cien. En cada uno que quieras usar Bootstrap simplemente tienes que enlazar el CSS (y el JS si necesitas componentes interactivos). 

<!-- Bootstrap actua sobre cualquier etiqueta a la que le pongas sus clases -->
<div class="container">         <!-- actua sobre este div -->
  <p class="text-primary">...</p>  <!-- y sobre este parrafo -->
  <button class="btn btn-danger">   <!-- y sobre este boton -->
</div>

<p>Este parrafo no tiene clases de Bootstrap, se queda con el estilo por defecto</p>
Nota: Bootstrap tambien aplica algunos estilos globales al documento entero nada mas cargarlo (reseteando margenes, cambiando el box-sizing, etc.). Esto se llama "Reboot" y puede hacer que algunas cosas cambien de aspecto aunque no hayas puesto ninguna clase.

¿Cómo se editan y modifican los ficheros?

Los ficheros de Bootstrap en si (el CSS y el JS) no se editan directamente. Lo que se hace es escribir el HTML con las clases de Bootstrap y si se quiere personalizar algo se añade CSS propio por encima, en un archivo separado o en una etiqueta <style>.

La clave esta en el orden de los enlaces: el CSS de Bootstrap tiene que ir primero, y el CSS propio despues. Asi las reglas propias sobreescriben las de Bootstrap cuando hay conflicto. 

<head>
  <!-- 1. Primero Bootstrap (base) -->
  <link href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.2/dist/css/bootstrap.min.css" rel="stylesheet">

  <!-- 2. Despues mi CSS propio (sobreescribe lo que quiera cambiar) -->
  <link rel="stylesheet" href="css/mis-estilos.css">
</head>

Por ejemplo, si quiero cambiar el color del boton primary:

/* En mi archivo mis-estilos.css */
.btn-primary {
  background-color: #e63946;  /* rojo en lugar del azul de Bootstrap */
  border-color: #e63946;
}

/* O puedo crear una clase nueva que no exista en Bootstrap */
.mi-seccion-especial {
  background-color: #f1faee;
  padding: 2rem;
  border-radius: 12px;
}
Consejo: Nunca edites directamente los archivos de Bootstrap (el bootstrap.min.css). Si los actualizas en el futuro perderas todos los cambios. Siempre usa un archivo CSS propio separado.

¿Cómo se referencian los ficheros desde otras aplicaciones?

Hay dos formas de referenciar Bootstrap desde un proyecto, dependiendo de si lo tienes descargado localmente o lo cargas desde internet:

Opción A: CDN (internet) — lo mas sencillo, no hay que gestionar archivos:

<!-- Se carga desde internet, no importa donde este el HTML -->
<link href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.2/dist/css/bootstrap.min.css" rel="stylesheet">

Opción B: Archivo local — cuando se ha descargado Bootstrap a la carpeta del proyecto:

<!-- La ruta es relativa al archivo HTML -->

<!-- Si el HTML esta en la raiz y el CSS en una subcarpeta css/ -->
<link rel="stylesheet" href="css/bootstrap.min.css">

<!-- Si el HTML esta en una subcarpeta y el CSS en otra -->
<link rel="stylesheet" href="../css/bootstrap.min.css">

<!-- Si el HTML y el CSS estan en la misma carpeta -->
<link rel="stylesheet" href="bootstrap.min.css">

Para el JS es exactamente igual, cambiando .css por .bundle.min.js y poniendolo antes del cierre de </body>.

Ojo con las rutas relativas: Si abres el HTML haciendo doble clic y las imagenes o archivos CSS no cargan, probablemente la ruta esta mal. Un truco para verificarlo es abrir la consola del navegador (F12) y mirar si hay errores en rojo que digan "404" o "Failed to load resource".

¿Cómo afecta Bootstrap a distintos tipos de elementos HTML?

Aunque Bootstrap funciona añadiendo clases a cualquier etiqueta, hay algunos elementos sobre los que actua de forma especial:

  • Contenedores (div, section, main...) — se les aplican clases de layout como container, row, col.
  • Tipografia (h1-h6, p, span) — clases de texto como text-primary, fw-bold, lead.
  • Formularios (input, select, textarea) — Bootstrap los estiliza con form-control para que queden uniformes.
  • Botones (button, a) — con las clases btn y btn-*.
  • Tablas (table) — con table, table-striped, table-bordered, etc.
  • Imagenes (img) — con img-fluid para que sean responsive y no se salgan del contenedor.

6. Sistema de Grid (cuadrícula)

Esta parte al principio me costó un poco entender pero una vez que le pillas el truco es muy util. Bootstrap divide la pantalla en 12 columnas y tu decides cuantas columnas ocupa cada elemento. Asi puedes hacer layouts de forma muy rapida.

Concepto básico

Siempre hay que seguir esta estructura: container > row > col. Si no pones el row las columnas no funcionan bien (me paso varias veces). La suma de columnas en una fila tiene que ser 12 o menos. 

<div class="container">
  <div class="row">
    <div class="col-6">Columna 1 (ocupa la mitad)</div>
    <div class="col-6">Columna 2 (ocupa la mitad)</div>
  </div>
</div>

La navbar es uno de los componentes que mas tiempo me llevo entender. Para que el boton de "hamburguesa" (el que aparece en movil) funcione hay que tener el JS cargado, si no no hace nada.

5.6 Modal (ventana emergente)

El modal es bastante util para mostrar informacion extra sin salir de la página. Lo mejor es que funciona sin escribir nada de JavaScript, solo con los atributos data-bs-toggle y data-bs-target.

<button class="btn btn-primary" data-bs-toggle="modal" data-bs-target="#miModal">
  Abrir modal
</button>

<!-- Estructura del modal -->

<div class="modal fade" id="miModal" tabindex="-1">

        <h5 class="modal-title">Título del modal</h5>

      </div>

      </div>
      <div class="modal-footer">

        <button class="btn btn-primary">Guardar</button>
      </div>
    </div>
  </div>
</div>

8. Diseño Responsive y Utilidades

Bootstrap tiene un monton de clases de utilidad que te ahorran tener que escribir CSS propio. Al principio puede abrumar la cantidad de clases que hay pero con la practica se van aprendiendo.

6.1 Clases de espaciado (margin y padding)

El formato es: {propiedad}{lado}-{tamaño}. Al principio parece confuso pero es bastante logico cuando lo entiendes.

  • Propiedad: m para margin o p para padding.
  • Lado: t arriba, b abajo, s izquierda, e derecha, x horizontal, y vertical. Si no pones nada aplica a todos los lados.
  • Tamaño: del 0 al 5. Tambien existe auto.
<div class="mt-3">Margin top 3</div>

10. Conclusiones

Despues de hacer este tutorial creo que he aprendido bastante sobre Bootstrap. No ha sido facil al principio, sobre todo el sistema de grid que me costó entender, pero una vez que le pilles el truco va bastante bien.

Lo que he conseguido

  • Entender que es Bootstrap y porque tiene sentido usarlo en lugar de escribir todo el CSS a mano.
  • Incluir Bootstrap en una pagina web usando el CDN (la descarga local no la he probado mucho).
  • Usar el sistema de grid de 12 columnas para hacer diseños basicos.
  • Usar los componentes mas comunes: botones, alertas, tarjetas, navbar, formularios y modales.
  • Crear una pagina de perfil juntando todo.

Alcance del tutorial

Este tutorial cubre un nivel basico-medio. He intentado explicar las cosas que mas se usan pero hay muchos componentes y opciones que no he llegado a ver, como por ejemplo:

  • Personalizar Bootstrap con Sass (esto es bastante avanzado y no lo he llegado a hacer).
  • Usarlo junto con frameworks como React o Vue.
  • Componentes que no he llegado a cubrir: offcanvas, toasts, accordions, tabs... (pendiente de explorar).
  • Crear temas propios o usar plantillas de otras personas.

Opinion personal

Bootstrap me ha parecido bastante util para hacer paginas rapidamente. La curva de aprendizaje es corta y en poco tiempo ya puedes tener algo decente. Lo que no me gusta tanto es que todas las webs hechas con bootstrap se parecen mucho si no personalizas nada, lo que le da un aspecto un poco generico. Tambien hay veces que para hacer algo especifico acabas sobreescribiendo tanto el CSS que te preguntas si no era mas facil hacerlo desde cero.

En general para proyectos rapidos o de clase creo que es una muy buena opcion.

Mi recomendacion: Para cualquier proyecto rapido o de clase Bootstrap es genial. Si el diseño es muy importante y quieres algo mas original, quizas merece la pena mirar Tailwind CSS, aunque eso ya es otra historia y tampoco lo he usado yo todavía.

Referencias y recursos que he usado

  • Documentación oficial de Bootstrap 5 — la mas completa aunque al principio cuesta leerla
  • W3Schools — Bootstrap 5 — muy util para buscar ejemplos rapidos
  • MDN Web Docs — Bootstrap
  • Bootstrap Icons — iconos gratuitos que van muy bien con bootstrap
  • Stackoverflow — para resolver dudas puntuales (sin esto no hubiera terminado el tutorial)
